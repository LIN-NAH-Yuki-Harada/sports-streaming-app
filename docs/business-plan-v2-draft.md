  - 連携 UX の摩擦が完全消滅、ユーザー獲得チャネルとしての信頼性が確立

#### B. 配信プロトコル TCP 化 (B 案) — 4G 環境下での画質安定の根本解決
**5月10日の試合配信で確認した課題**: WebRTC (UDP) は 4G 上り変動下で画質が不安定。BAND (NAVER) など他プラットフォームは TCP 系プロトコルで再送機構が効くため安定。

LIVE SPOtCH もこれを実現するため、**配信者ブラウザの MediaRecorder で動画チャンクを生成し、中継 server (Cloudflare Workers / Fly.io) 経由で RTMP に変換** する方式に切り替えます。

|---|---|
| 工数 | 5-7 日（Claude 主導の AI 駆動開発） |
| 月コスト | +$5-20（中継 server, 約 ¥800-3,000）|
| 効果 | 4G 体育館でも 1080p / 3-5 Mbps の高画質安定配信 |
| マイルストーン | 5/15 設計完了 / 5/22 実装完了 / 5/29 実機検証 / 5/31 本番反映 |

#### C. 残クリティカルバグ完全対応

## 5. Exit シナリオ (Appendix)

### 5.1 スポーツブル買収戦略

スポーツブル (KDDI 系・運営: ベースボール・マガジン社等が出資) は地域大会・部活配信を扱っていない。理由:
- プロカメラクルー派遣ベースで運営コストが高い
- ローカル試合の収益性が合わない

LIVE SPOtCH は **UGC (保護者撮影) でこのコスト構造を解消**。買収側メリット:
- 撮影者の負担減 (保護者が UGC として提供)
- 運営コスト大幅削減
- ユニークユーザー数の大幅 UP (ローカルスポーツは膨大な未開拓市場)

### 5.2 接触ロードマップ
- Year 1 中盤 (2026年秋): 連盟アプローチを通じてスポーツブル運営側との接点形成
- Year 2 (2027年): 本格交渉開始
- Year 3 以降: 買収提案 / 戦略提携 のいずれか

> ※本 Exit は事業継続前提の「もう一つの出口」であり、融資返済は本業 (LIVE SPOtCH サブスク) で確保いたします。
